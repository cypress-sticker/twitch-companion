# Contributing to コアたん

コントリビューションありがとうございます！バグ報告・機能提案・プルリクエストを歓迎しています。

---

## バグ報告・機能提案

[Issues](https://github.com/tky1192tky/twitch-companion/issues) からお気軽にどうぞ。  
以下を書いてもらえると助かります：

- **バグ報告**: 再現手順、期待する動作、実際の動作、環境（OSバージョン、アプリバージョン）
- **機能提案**: やりたいこと、なぜ欲しいか

---

## プルリクエスト

### 事前準備

1. リポジトリをフォーク
2. 開発環境をセットアップ（[README.md](README.md) の「開発環境のセットアップ」参照）
3. 自分のTwitch Appを用意して `.env` に `TWITCH_CLIENT_ID` を設定

### ブランチ運用

```
main        ← 安定版（リリース済み）
feature/*   ← 新機能
fix/*       ← バグ修正
```

### コミットの作り方

機能・修正ごとに小さくコミットしてください。  
コミットメッセージは `feat:` / `fix:` / `chore:` などのプレフィックスをつけると読みやすいです。

### PRを出す前に

- `npm start` で動作確認
- 既存の機能に影響がないか確認（特にアラート・定期コメント・OBSオーバーレイ）

---

## コードの方針

- フレームワークを増やさない方向で（現状 Electron + tmi.js + ws のみ）
- UIは `renderer/` 以下の素のHTML/CSS/JSで完結させる
- 設定はすべて `userData/settings.json` に保存する設計を維持する
